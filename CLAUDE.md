# デジトレ開発のルール（Dockerの箱の中で動くClaude向け）

このフォルダは Docker の箱の中で、**権限フル委任モード（`--dangerously-skip-permissions`）**で
開発されることがあります。確認なしで作業を進められる分、事故を防ぐため
以下のガードレール（＝安全ルール）を必ず守ってください。

## 1. 削除は「退避」で行う（完全削除しない）

- ファイルを削除するときは完全に消さず、`_trash/(今日の日付)/` フォルダを作ってそこへ移動する。
- 完全な削除はユーザーが自分で行う。

## 2. GitHub の履歴を壊さない

- `git push --force`（強制プッシュ）や、過去の履歴を書き換える操作は**禁止**。
- GitHub 上のコピーは「最後に戻れる安全な場所」なので、絶対に上書き・破壊しない。

## 3. コミットは小さく・日本語で

- 意味のあるまとまりごとに、こまめに小さくコミット（セーブ）する。
- コミットメッセージは日本語で「何をしたか」が一目で分かるように書く。
- 大きな変更を一度のコミットに詰め込まない（後から差分を読めなくなるため）。

## 4. 説明・コメントは日本語で

- コードのコメントや、行った作業の説明は日本語で書く。
- ユーザーは非エンジニアなので、専門用語には短い解説（＝…）を添える。

## 5. 破壊的な操作の前には理由を残す

- 大量に消す・大きく作り替えるときは、「何を・なぜ」やるのかを
  出力やコミットメッセージにはっきり残してから実行する。

---

これらは、確認を取れない自動モードでも安全に開発を進めるための最低限のルールです。
