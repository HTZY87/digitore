# デジトレ 使い方ガイド

科学的根拠にもとづくAIパーソナルトレーナー「デジトレ」。
体の情報から1日の栄養目標を計算し、食べたものを記録して、PFC（タンパク質・脂質・炭水化物）管理を助けます。

くわしい設計は [SPEC.md](SPEC.md) を参照してください。

---

## 📱 iPhoneで使う（PWA版）

サーバー不要・完全無料で、iPhoneのホーム画面から起動できる版です。
計算も記録もすべて端末の中で行われ、データは外に出ません。

### 入れ方（初回だけ）

1. iPhoneの Safari で **https://htzy87.github.io/digitore/** を開く
2. 共有ボタン（□に↑）→「**ホーム画面に追加**」をタップ
3. ホーム画面にできた「デジトレ」アイコンから起動
   （一度開けば、電波がない場所でも使えます）

### データの注意

- 記録は **iPhoneの中だけ** に保存されます（PC版の records.csv とは別）。
- Safariのデータ消去などで記録が消えることがあるため、
  「一覧」画面の **CSVエクスポート** で、ときどき控えを取ってください。
  取った控えは同じ画面の **取り込み（インポート）** で戻せます。

### PCで動きを確認したいとき（開発用）

```
python -m http.server 8000 --directory docs
```
→ ブラウザで http://127.0.0.1:8000/ を開く。
テストは http://127.0.0.1:8000/test.html （開くと自動で19件走ります）。

## 🌐 Web版（ブラウザで使う）の始め方

### 前提
- お使いのPCに **Docker Desktop** が入っていること。
- このフォルダ（`docker-compose.yml` があるフォルダ）を開けること。

### 手順（あなたのPCのターミナルで実行）

> ⚠ 注意：箱（コンテナ）を作り直すと、もし箱の中でClaude Codeを動かしていた場合は
> その会話がいったん切れます。コードはすべて保存済みなので、消える心配はありません。

**1. このフォルダに移動する**（例。実際の置き場所に合わせてください）
```
cd デジトレのフォルダ
```

**2. 箱を作り直す**（Webの公開設定 `ports` を反映するため、初回はこれが必要）
```
docker compose down
docker compose up -d --build
```

**3. Web版アプリを起動する**
```
docker compose exec claude python app.py
```
画面に `Running on http://...:5000` と出れば起動成功です。
このターミナルは開いたままにしておきます（止めるときは Ctrl + C）。

**4. ブラウザで開く**

→ **http://127.0.0.1:5000/**

これでメニュー画面が出ます。「初期設定」→「食事を記録」→「グラフ」の順に使えます。

```

```

## 📊 グラフの日本語フォントについて

グラフに日本語を表示するためのフォントは、すでに `assets/fonts/` に用意済みです。
**ふだんは何もしなくて大丈夫です。**

別のPCに git clone して動かす場合など、フォントが無いときだけ一度だけ実行してください：
```
python setup_font.py
```
（フォントが無い環境では、グラフの文字は自動で英語に切り替わります）

## 📁 ファイルの役割（ざっくり）

| ファイル | 役割 |
|---|---|
| `app.py` / `templates/` | ⑦ Web版の入口と画面 |
| `docs/` | ⑧ iPhone対応のPWA版（ブラウザだけで動く別実装） |
| `main.py` | ターミナル版の入口 |
| `targets.py` | ② 目標カロリー・PFCの計算 |
| `intake.py` / `foods.csv` | ③ 食べたものの合計（食品データ） |
| `consult.py` | ④ 残り許容量（目標 − 実績） |
| `record.py` | ⑤ 記録（records.csv に保存） |
| `visualize.py` / `setup_font.py` | ⑥ グラフ作成・フォント用意 |
| `safety.py` | ⑥ 安全警告（極端な設定への注意） |

## 🔒 データについて

- あなたの個人データ（`profile.json`・`records.csv`）と、作ったグラフ画像、フォント本体は
  GitHub には上げない設定（`.gitignore`）にしています。手元にだけ残ります。
- このアプリは「計算・記録は本物のコード／相談はAI」という方針で作っています。
  数値がブレないよう、計算はすべてコードが担当します。
